# `docs/` — your published website

This folder is the **rendered output** of [`reports/`](../reports/). GitHub Pages serves it as your project's public site.

## Don't edit files here by hand

Every file in this folder is regenerated by `quarto render reports/`. If you change `index.html` directly, your edits will be overwritten the next time someone renders.

Edit the `.qmd` sources in [`reports/`](../reports/) instead.

## Enable GitHub Pages

1. Push your changes to GitHub.
2. Repo → **Settings** → **Pages** → **Build and deployment** → **Deploy from a branch**.
3. Branch: `main`, Folder: `/docs`. Save.

Your site will be at `https://<your-username>.github.io/<your-repo>/`.

## Why `docs/` and not `_site/`?

GitHub Pages can serve from `/` or `/docs` on a branch without extra config. We render to `/docs` so you don't need a separate `gh-pages` branch and the site stays in sync with the code on `main`.
